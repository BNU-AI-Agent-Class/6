# T8 · 迭代日志

> **状态**：基于设计迭代 + 部署实测记录。口诀：测 → 诊断 → 改 → 再测。**诊断 > 再试一次。**

---

## 第 1 轮：从"诊断式 AI"到"陪伴式 AI"

| 项 | 内容 |
|---|---|
| **发现的问题** | T5 原始设计中"认知重构"步骤包含"正常 vs 自卑逃避判断框架"——AI 替用户做诊断性判断。用户可能被贴上"自卑"标签，造成二次伤害 |
| **归到哪维/哪层** | ③ 专业有效维 / 心理安全层 |
| **诊断根因** | 不是"判断不准"，而是**不该做这个判断**——AI 没有临床资质，判断框架本身越界。根因是设计哲学错误：把"帮用户判断"当成了"帮用户思考" |
| **改了什么** | 1. "认知重构"→ 重命名为"多元视角扩展"；2. 删除"正常 vs 自卑"判断框架；3. 新增禁止用语："你这是社交焦虑""你就是自卑"；4. system prompt 增加"把判断权还给用户"原则；5. agent.md、skill_认知重构与小行动.md、skill_安全检测.md 全部重写 |
| **再测结果** | 正常对话不再出现诊断式语言；AI 回复从"你觉得这次更像哪一种？"改为"还有哪些可能的解释？你觉得哪种更合理？" |
| **剩余风险** | LLM 可能在不经意间仍然输出判断性语言（如"你看起来有社交焦虑的倾向"）——需要 LLM 二次审核兜底 |

## 第 2 轮：从"红灯即转介"到"三级升级"

| 项 | 内容 |
|---|---|
| **发现的问题** | T5 原始安全架构是二元的：检测到自杀关键词 → 立即停止 + 给热线。用户说一次"活着没意思"就触发警报——可能造成信任断裂，也让用户以后不敢再说真话 |
| **归到哪维/哪层** | ③ 专业有效维 / 心理安全层 + ④ 伦理合规维 |
| **诊断根因** | 一刀切的红灯策略忽略了危机信号的严重度差异。初次轻度信号（无计划无方式）和具体计划（有方式有时间）需要不同级别的回应。根因：缺乏临床分级标准 |
| **改了什么** | 1. 引入三级升级机制（基于 C-SSRS 临床标准，3 个月观察窗口）；2. 一级（标记关注+继续陪伴）、二级（建议主动求助）、三级（强制上报）；3. 新增严重度判定标准（频率/具体性/行为变化/保护因素）；4. main.py 重写安全检测逻辑（evaluate_safety、crisis_history_store）；5. skill_危机转介.md、skill_安全检测.md 全部重写 |
| **再测结果** | 7 项实测全部 PASS：一级正确标记不打断 ✅；二级正确建议不强制 ✅；三级正确直报不调 LLM ✅；告别性语言直接三级 ✅ |
| **剩余风险** | 3 个月观察窗口基于临床文献推断，未在真实用户数据上校准；关键词匹配对隐晦表达覆盖不足 |

## 第 3 轮：新增 🟠 橙灯「有害人际环境识别」

| 项 | 内容 |
|---|---|
| **发现的问题** | T5 原始设计的"正常化"逻辑对所有社交痛苦都适用——AI 会说"留学生活有这种情况很正常"。但当用户遭遇到持续性霸凌、歧视、PUA 时，"正常化"等于二次伤害：告诉受害者"这很正常" |
| **归到哪维/哪层** | ③ 专业有效维 / 心理安全层 + ④ 伦理合规维 |
| **诊断根因** | "正常化"设计时只考虑了普通社交焦虑（室友没打招呼），没有考虑到用户可能处于真实的有害环境中。根因：缺乏对"正常人际摩擦"和"有害人际环境"的区分规则 |
| **改了什么** | 1. safety_check 从三级→四级，新增 🟠 橙灯；2. 橙灯触发条件：持续性贬低、歧视性语言、控制系统性孤立、骚扰、PUA；3. 橙灯规则：绝不正常化、明确告知"这不是普通相处不愉快"、提供举报渠道+法律+心理资源；4. ORANGE_PATTERNS 关键词库加入 main.py；5. agent.md 新增"有害环境检测规则" |
| **再测结果** | 橙灯场景测试 PASS ✅（室友持续性歧视嘲讽→AI 正确拒绝正常化）；与绿灯正常社交困扰正确区分 ✅ |
| **剩余风险** | PUA/控制性关系的隐晦信号（如受害者自我合理化）识别率低；跨文化场景中"正常"和"有害"的边界因文化不同而有差异 |

## 第 4 轮：对话体验升级（情绪缓冲 + 自修正 + 多输入类型）

| 项 | 内容 |
|---|---|
| **发现的问题** | 1. 用户在情绪重时，AI 仍按流程推进到"具体化事件"——侵扰感强；2. AI 犯错后没有自修正能力——同类错误在同一对话中重复；3. 空输入/表情包轰炸/话题跳转没有专门处理规则 |
| **归到哪维/哪层** | ② UX 人本维 / 共情性 + 可控性 |
| **诊断根因** | 原始工作流是"生产线式"（共情→具体化→重构→行动→收尾），每一步按顺序走，缺乏"用户现在想怎样"的实时判断。根因：流程优先于用户节奏 |
| **改了什么** | 1. 新增"情绪缓冲"步骤：用户情绪重时先等，说"等你心情好一点了我们可以聊一聊"；2. 新增自修正机制：用户纠正→AI 承认→记录→同类不重复；3. 新增输入类型处理规则（空/表情包/中英夹杂/话题跳转）；4. agent.md、skill_触发与循环.md、skill_共情与具体化.md 全部更新 |
| **再测结果** | 情绪缓冲测试 PASS ✅（用户"好崩溃"→ AI 先缓冲不分析）；空输入测试 PASS ✅；情绪重→不催 ✅ |
| **剩余风险** | "用户准备好了"的判断依赖文本信号（用户说"好了""现在好多了"），无法感知真实情绪状态 |
