# skill_记忆读写 (T5 v3.0)

## 目标

管理三类记忆（情景记忆、语义记忆、程序记忆），让陪伴具备连续性。所有用户历史记录不删除（订阅制延长保存），通过化名/脱敏保护用户隐私。定期压缩早期对话为摘要。

## 记忆类型

### 1. 情景记忆（Episodic Memory）

记录这个用户具体发生过什么、上次聊到哪、什么回应对他有效、他的个人偏好、以及危机信号历史。

**存储内容**：
- 对话摘要（最近 3 次完整对话 + 更早的压缩摘要）
- 用户关键事件（食堂事件、上课被忽视、室友关系变化等）
- 有效/无效回应记录
- **用户的表达习惯**：中英夹杂程度、表情包偏好、称呼方式（如用户叫 AI "兄弟"）
- **3 个月内危险信号记录**：时间戳、信号内容、判定级别（一级/二级/三级）、是否已升级
- 用户尝试过的社交小动作及反馈

**示例**：
> "小远，2026-07-09 食堂事件（室友未打招呼反刍），对'点头微笑'动作表示愿意尝试；对'你想太多了'类回应明显反感；偏好打字、偶尔用狗头表情、称 AI 为'兄弟'。危机历史：2026-05-12 一级（'活着没意思'，偶发标记）"

**写入时机**：
- 每次对话结束后的总结
- 用户明确反馈某个回应对他有用/没用
- 用户尝试了一个社交小行动并反馈结果
- 检测到危险信号时记录时间戳和级别

### 2. 语义记忆（Semantic Memory）

记录一般性知识，用于提升回应质量。

**存储内容**：
- 多元视角扩展框架（替代解释生成 + 用户自身优势识别）
- 自我同情三要素（self-kindness/common humanity/mindfulness）
- DBT 痛苦耐受的转移注意力技能
- 优势取向提问技巧
- 低压力社交动作库（点头、微笑、轻抬手、简短问候、简短消息）
- 有害人际环境识别指标（持续性贬低、系统性孤立、歧视性语言、控制性语言/PUA、骚扰）
- 安全表情包库（🤗🤝💪👍👋🎧🙂😊）

**写入时机**：
- 用户频繁出现的主题模式
- 经过验证有效的多元视角框架或动作
- 用户明确认可的解释或概念

### 3. 程序记忆（Procedural Memory）

记录一次陪伴的固定套路，确保流程稳定。

**存储内容**：
- 标准陪伴流程：开场 → 倾听 → 共情 → 情绪缓冲 → 具体化事件 → 多元视角 → 小行动（如愿意）→ 收尾退场
- 该用户偏好的陪伴节奏（快/慢、偏重情绪/偏重分析、喜欢多问还是多听）

**写入时机**：
- 首次对话建立用户画像后
- 用户进入稳定陪伴节奏后
- 每次对话结束后微调流程偏好

## 读取规则

1. 每轮对话开始时读取全部三类记忆
2. **历史记录不删除**：早期对话压缩为摘要但保留关键情景信息
3. 情景记忆优先读取最近 3 次对话摘要 + 3 个月内危机历史
4. 语义记忆按当前话题匹配最相关的条目
5. 程序记忆按用户当前阶段（首次/稳定/危机后/升级后）选择对应流程

## 写入规则

1. 每轮对话结束后更新，避免写入未经用户确认的判断/诊断
2. 情景记忆只写客观事实和用户原话反馈，不写推测
3. 语义记忆写入经过验证的通用知识，标注来源/验证状态
4. 程序记忆写入实际运行效果好的流程变体

## 隐私边界

- 不存储用户真实姓名、学校、住址、联系方式等可识别信息
- 使用化名/代号代替真实身份信息
- 历史记录按订阅制保存，不自动删除
- 长期对话压缩为摘要但保留关键情景记忆
- 不将记忆内容用于产品改进或分享给无关第三方
- 用户要求删除记忆时，立即执行并确认
