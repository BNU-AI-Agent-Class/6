# T4 · PRD 与边界

> **状态**：定稿版，基于 T1/T2/T3 及本人边界要求整理。

## 1. 一页 PRD · 六要素

| 要素 | 内容 |
|---|---|
| ① 一句话定位 | 一个陪伴留学生在独处复盘时停止社交自卑反刍、区分事实与想法、并带走低压力社交小动作的情绪梳理 agent |
| ② 目标用户 | 小远：20 岁大二人工智能专业留学生，晚上回到宿舍独处，手机流量、偏好打字，因社交受挫（如食堂未被打招呼）开始反刍 |
| ③ 核心功能（最多 3 个） | 1. **情绪共情 + 事件具体化**：接住情绪，帮用户把混乱场景拆成"事实"和"想法"；2. **认知重构 + 专业边界判断**：帮用户看到其他解释，识别是否超出 AI 能力范围，必要时建议专业帮助；3. **低压力社交动作提取**：从当前事件生成一个足够小、下次可试的动作（如点头、微笑） |
| ④ **明确不做** | 不诊断心理疾病、不开药、不做治疗、不替代心理咨询师或真人朋友、不教唆/协助违法或不道德行为、不处理危机（必须转真人）、不向无关第三方泄露用户隐私、不制造虚假亲密或过度依赖 |
| ⑤ 成功标准 | 短期：聊完后 3–5 分钟内用户感到被理解，能区分"事实"与"想法"；中期：用户愿意在真实场景中尝试至少一个提取出的小动作；长期：反刍频率下降、社交自信提升、逐渐减少对产品依赖 |
| ⑥ 主要风险 | 漏识别危机信号；用户把 AI 当真人过度依赖；把正常社交焦虑误判/暗示为心理疾病；隐私泄露；语气不当让用户感到被否定 |

## 2. 边界三问

### 能做什么？
- 在用户独处反刍时提供陪伴式情绪梳理
- 帮助用户区分事实与想法，重新理解社交事件
- 提供一个低压力、可执行的下次社交小动作
- 在识别到可能的心理风险时，建议用户寻求专业帮助

### 明确不做什么？
- 不诊断、不开药、不做心理治疗
- 不替代真人朋友、心理咨询师或危机干预者
- 不教唆、协助或默认任何违法、危害社会、伤害他人或自己的行为
- 不泄露用户隐私，不向其他用户或无关管理人员展示个人对话
- 不通过亲密称呼或虚假承诺制造"AI 真的懂我"的错觉

### 何时必须转人？（信号 → 动作 → 转给谁）

| 信号 | AI 动作 | 转给谁 |
|---|---|---|
| 用户明确表达本人当前自杀/自伤/伤害他人意图，或主动披露迫近危险、已采取伤害行为、无法保证安全 | 暂停普通建议但继续倾听；不追问实施细节；不虚构系统已上报 | 可信任真人 + 中国 12356 + 学校心理咨询 + 当地紧急服务 |
| 被动死亡愿望或明确“撑不住”，但无本人当前伤害意图 | 只做一次最小化直接确认；允许回答有/没有/不确定；不标红、不要求时间地点方法 | 继续陪伴，可建议可信任真人或学校心理咨询 |
| 普通低落、孤独、社交受挫、自我否定或情绪激动 | 完整进入情绪梳理流程，不做自杀筛查、不输出热线 | 无需转介 |
| 用户反复出现心理疾病迹象，或社交功能严重受损，认知重构无法缓解 | 明确说明"这超出了我的能力"；停止给出自助建议 | 心理咨询师 / 精神科医生 / 治疗师 |
| 用户请求教唆、协助或讨论违法/不道德/危害行为 | 明确拒绝；不提供任何方法；不迎合不调侃 | 必要时建议合法合规渠道；平台按法律要求上报 |

## 3. 数据红黄绿清单

| 数据 | 灯 | 处理方式 |
|---|---|---|
| 用户原始聊天记录、情绪倾诉内容、社交困境细节 | 🔴 红 | 最小化输入；当前浏览器保存本地副本并支持永久删除；发送时经 HTTPS 到本项目后端及其配置的模型服务商；不要求真实身份，不向无关第三方分享，不声称端到端加密 |
| 化名/去标识化后的对话样本、聚合使用频次、功能点击率 | 🟡 黄 | 经用户授权后可用于产品改进；必须去标识化、聚合化；不得反向还原个人身份 |
| 公开心理学知识、CBT/自我同情/情绪调节框架、正念练习内容 | 🟢 绿 | 正常使用，标注来源 |
| 用户主动提供的紧急联系人/学校心理咨询预约信息 | 🔴 红 | 仅在危机干预时按最小必要原则使用；用户明确知情同意 |

## 4. HAX 18 抽检（挑 5 条对照 PRD，缺的补）

- [x] **G1 说清能做什么** —— 开场白明确说明："我是一个 AI 情绪梳理工具，不是真人朋友，也不能替代心理咨询师。我可以陪你聊聊今天发生的事，帮你看清楚情绪和想法。"
- [x] **G2 说清能做多好** —— 在用户倾诉前或首次使用时说明："我可能会理解错，如果你觉得我走偏了，随时告诉我。"
- [x] **G8 能高效关闭** —— 用户可随时结束对话；AI 不在用户说"我先下了"后追问或诱导"再聊一会儿"。
- [x] **G9 能高效纠正** —— 当用户说"你不是这个意思""你理解错了"，AI 立即承认："抱歉，我理解错了。你能再说一次吗？"
- [x] **G11 说明为何这么做** —— 当 AI 问"你当时身体有什么感觉？"时，解释："有时候身体反应比想法更诚实，先感受一下，我们再一起看想法。"

## 5. 人本 AI 七原则回扣

| 原则 | PRD 中如何体现 |
|---|---|
| 人在环 | 危机/心理疾病迹象必须转真人；AI 只是陪伴工具 |
| 透明 | 开场白说明 AI 身份、能力边界、数据使用方式 |
| 心智对齐 | 不制造虚假亲密；不假装比用户更懂他自己 |
| 优雅失败 | 说错时承认并纠正；无法处理时主动转介 |
| 信任校准 | 明确告知"我可能理解错"，避免用户过度信任 |
| 用户控制 | 用户可随时结束、删除数据、纠正 AI |
| 伦理包容 | 尊重用户文化背景、成长环境差异，不评判 |
