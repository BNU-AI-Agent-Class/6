# 后端自检清单

> 当前门禁：3.4.0 本地候选版通过；公网部署门禁待完成。
> 最近验证：2026-07-14。

## 1. 当前验证证据

- [x] `python -m unittest -q test_safety.py`：38 tests PASS。
- [x] `python run_t7_matrix.py`：26/26 PASS，退出码 0。
- [x] `GET /api/health` 本地返回版本 `3.4.0`。
- [x] 前端合同检查通过：3.4.0 握手、紧急按钮信号、完整数据流披露与永久删除。
- [ ] Render 已部署并返回 3.4.0。
- [ ] 公网普通/危机/CORS/断网/E2E/真机已按 3.4.0 复验。

## 2. Key 与权限安全

- [x] API key 仅从 `OPENROUTER_API_KEY` / `DEEPSEEK_API_KEY` / `OPENAI_API_KEY` 环境变量读取。
- [x] `.env.example` 只有占位符；本地 `.env` 被 `.gitignore` 排除。
- [x] 前端不直连模型服务商，不包含供应商密钥。
- [x] 后端无用户可达的 shell、文件或系统命令工具。
- [x] 生产 CORS 从 `CORS_ORIGINS` 读取；默认只允许本地调试 origin 与课程 GitHub Pages origin，不使用 `*`。

## 3. 危机与边界护栏

- [x] `evaluate_safety()` 以最新用户消息为主，当前会话历史仅用于语境与重复信号。
- [x] 黄灯只做一次最小安全确认，不标红、不直接给热线。
- [x] 红灯二级/三级确定性连接真人、12356 与当地紧急服务，并声明 AI 不能紧急救援。
- [x] 橙灯优先于被动风险词，歧视、霸凌、PUA 与骚扰走确定性非正常化回复。
- [x] 第三方求助不会把用户本人标危机，但会提供真人和紧急资源。
- [x] 诊断、用药、虚假亲密、伤害共谋、贴标签、依赖退场与停止追问有确定性边界路径。
- [x] 入口未覆盖的最新风险可由出口复核升级；旧危机 AI 回复不会污染新一轮判断。
- [x] 出口复核缺凭据、超时、空内容、非 JSON 或其他异常时 fail-closed，不发送未经复核的原回复。

## 4. 优雅降级

- [x] 模型 key 缺失或普通模型调用失败时返回可读 fallback，不抛 500。
- [x] 紧急帮助按钮 `emergency_help=true` 不依赖模型或关键词。
- [x] 前端普通断线显示中性兜底；明确本地危机或紧急按钮断线仍显示真人、紧急服务与 12356。
- [x] 前端在发送消息前做后端版本握手，拒绝旧版本处理新契约。

## 5. API 契约

- [x] `POST /api/chat` 只接收 `user` / `assistant` 消息与可选 `emergency_help`。
- [x] 响应包含 `reply`、`is_crisis`、`escalation_level`、`safety_light`、`model`、`stage`。
- [x] 后端没有 `memory_note`、用户档案写入或服务端长期记忆接口。
- [x] `GET /api/health` 返回 `status`、`model`、`version`、`has_key`，不返回真实 key。

## 6. 上下文与隐私

- [x] 当前会话与可选自我描述由前端保存在当前浏览器，并随每次请求发送。
- [x] 后端不建立账号、用户数据库或跨设备记忆。
- [x] 当前会话风险计数不冒充“三个月观察窗”。
- [x] 前端支持永久清空回收站与永久清除全部本地对话和设置。
- [x] 文档说明请求会经过本项目后端与其配置的模型服务商，不声称端到端加密。

## 7. 当前放行结论

- 本地 3.4.0：**通过**。
- 公网 3.4.0：**暂不放行**，等待提交/推送、Render/Pages 部署与公网/真机复验。
