# 密钥环境变量清单

> 项目：留学生情绪梳理 Agent  
> 最终平台：Render  
> 更新日期：2026-07-13  
> 原则：只记录变量名、用途和验证状态，**绝不记录真实密钥值**。

## 1. Render 生产变量

| 变量名 | 是否敏感 | 用途/建议值 | 当前验证状态 |
|---|---:|---|---|
| `OPENROUTER_API_KEY` | **是** | OpenRouter 调用凭据，仅供后端读取 | 已生效；公网健康检查返回 `has_key=true` |
| `CORS_ORIGINS` | 否 | `https://bnu-ai-agent-class.github.io` | 已生效；浏览器预检返回该 origin |
| `LLM_MODEL` | 否 | `deepseek/deepseek-v4-flash` | 已生效；健康检查返回该模型 |
| `LLM_BASE_URL` | 否 | `https://openrouter.ai/api/v1` | 可显式配置；当前公网对话已成功调用模型 |
| `PORT` | 否 | Render 自动注入 | **不要手工填写** |

`CORS_ORIGINS` 填 origin 即可，不含 `/6/` 路径，也不加末尾斜杠。

## 2. 可选兼容变量

| 变量名 | 是否敏感 | 何时使用 |
|---|---:|---|
| `DEEPSEEK_API_KEY` | **是** | 切换为 DeepSeek 官方兼容 API 时 |
| `OPENAI_API_KEY` | **是** | 切换为 OpenAI 兼容 API 时 |
| `MODEL` | 否 | 兼容旧配置；优先使用语义更清楚的 `LLM_MODEL` |
| `PYTHON_VERSION` | 否 | 需要固定运行时版本时再配置；当前构建日志使用 Python 3.14.3 |

后端密钥选择优先级为：`OPENROUTER_API_KEY` → `DEEPSEEK_API_KEY` → `OPENAI_API_KEY`。

## 3. Render 填写位置

1. 打开 Render 服务 `bnu-agent-6`。
2. 进入 **Environment**。
3. 逐项添加变量名和值并保存。
4. 不要添加 `PORT`；启动命令直接读取 `$PORT`。
5. 变量保存并重新部署后，访问 `https://bnu-agent-6.onrender.com/api/health`。
6. 只确认 `has_key=true`，不要截图或复制真实密钥值。

## 5. 安全与轮换规则

- `.env`、`.env.local` 和任何导出的生产变量文件不得提交 Git。
- 不在前端 JavaScript 中保存任何供应商密钥；前端只能调用 Render 后端。
- 若密钥曾出现在 Git、截图、聊天或公开日志中，立即在供应商控制台作废并新建；删除文件不能替代轮换。
- 生产与个人实验建议使用不同密钥，并在 OpenRouter 侧设置额度或速率限制。
- 交作业只提交本清单，不提交任何真实值。
