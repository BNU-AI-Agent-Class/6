# 密钥环境变量清单

> 项目：留学生情绪梳理 Agent  
> 平台：Railway  
> 原则：只记录变量名、用途和配置状态，**绝不记录真实密钥值**。

## 1. 生产必配

| 变量名 | 是否敏感 | 用途 | Railway 当前状态 |
|---|---:|---|---|
| `OPENROUTER_API_KEY` | 是 | 调用 OpenRouter；只由后端读取 | 本地 `.env` 已配置；Railway Variables 待配置 |
| `CORS_ORIGINS` | 否 | 限制允许访问后端的浏览器 origin | 填 `https://bnu-ai-agent-class.github.io` |

## 2. 建议显式配置

| 变量名 | 是否敏感 | 建议值/用途 | 当前状态 |
|---|---:|---|---|
| `LLM_MODEL` | 否 | `deepseek/deepseek-v4-flash`，避免依赖隐式默认值 | 已于 2026-07-12 核对 OpenRouter 当前模型目录；Railway 待配置 |
| `LLM_BASE_URL` | 否 | `https://openrouter.ai/api/v1`；代码也会按 key 自动推导 | 可选，建议显式配置 |
| `PORT` | 否 | Railway 自动注入，启动命令读取 `$PORT` | **不要手工填写** |

## 3. 替代供应商变量（与 OpenRouter 三选一）

| 变量名 | 是否敏感 | 何时使用 | 备注 |
|---|---:|---|---|
| `DEEPSEEK_API_KEY` | 是 | 改为 DeepSeek 官方兼容 API 时 | 不要与不再使用的供应商 key 同时长期保留 |
| `OPENAI_API_KEY` | 是 | 改为 OpenAI 兼容 API 时 | 配合 `LLM_BASE_URL`、`LLM_MODEL` 核对 |
| `MODEL` | 否 | 兼容旧配置时 | 优先使用语义更清楚的 `LLM_MODEL` |

密钥选择优先级由代码定义为：`OPENROUTER_API_KEY` → `DEEPSEEK_API_KEY` → `OPENAI_API_KEY`。

## 4. Railway 填写规则

1. 打开 Railway Service → Variables。
2. 新建 `OPENROUTER_API_KEY` 并粘贴真实值；不要在聊天、截图、提交记录或 Markdown 中展示。
3. 新建 `CORS_ORIGINS=https://bnu-ai-agent-class.github.io`；origin 不含 `/6/` 路径，也不加末尾斜杠。
4. 新建 `LLM_MODEL=deepseek/deepseek-v4-flash`。
5. 建议新建 `LLM_BASE_URL=https://openrouter.ai/api/v1`。
6. 不要新建 `PORT`，由 Railway 自动注入。
7. Variables 变更会触发部署；部署完成后访问 `/api/health`，确认 `has_key=true`。

## 5. 轮换与泄漏处理

- 若密钥曾出现在 Git、前端、公开日志或截图中：立即在供应商控制台作废并新建，不以“删掉文件”代替轮换。
- `.env`、`.env.local` 和任何下载的生产变量文件不得提交。
- 生产与个人实验建议使用不同 key，并设置 OpenRouter 侧额度/速率限制。
- 交作业时只提交本清单，不提交任何带真实值的环境文件。
