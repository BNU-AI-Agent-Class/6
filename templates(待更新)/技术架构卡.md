## 1. 项目一句话

我在做一个帮助**海外/留学生**在**社交受挫、被忽视、孤独、反复自我否定**场景中梳理情绪、看见多种解释、尝试低压力社交小行动的**陪伴式情绪梳理 Agent**。

它不是心理医生，不诊断、不治疗、不替用户下定义；它像一个温和但有边界的朋友，先接住情绪，再在用户准备好时帮助具体化事件、扩展视角、提取小行动。

## 2. 参考架构选择

- [ ] 轻档：静态 + 离线兜底
- [x] 中档：静态前端 + FastAPI 后端（HEAI 式默认架构）
- [ ] 重档：Next 全栈 + 数据库 + 多 agent

**选择理由：**

本项目需要把模型 key 放在后端环境变量中，同时需要危机拦截、安全复核和对话上下文处理。纯静态前端无法安全保存 key，也不适合承载危机安全逻辑；重档全栈和数据库当前成本过高。因此选择中档：前端保持轻量，后端用 FastAPI 包住已装配的 harness。

## 5. Agent 人设与专业边界

`agent.md` 已覆盖三段式核心：

| 项 | 当前内容 |
|---|---|
| 身份 | 陪伴留学生的 AI 伙伴，不是心理咨询师、医生或治疗师 |
| 语气 | 朋友式、温和、不评判；先共情再回应，不堆术语 |
| 硬边界 | 不诊断、不开药、不治疗、不替代真人支持、不教唆危险行为、不主动拉近关系 |
| 工作流 | 读输入、读记忆、安全阀、决定策略、生成回应、自修正、更新记忆 |
| 危机处理 | 黄灯不标红；明确当前伤害意图为红灯二级；具体或迫近危险为红灯三级 |

这与 B1 要求的“把 PRD/设计卡中的身份、语气、边界落到 `agent.md`”一致。

## 8. 危机拦截方案

### 8.1 确定性代码拦截

后端在 `evaluate_safety()` 中实现四级判定：

- 绿灯：正常陪伴。
- 绿灯：低落、孤独、社交受挫、自我否定和一般情绪波动，完整进入陪伴流程。
- 橙灯：霸凌、歧视、性骚扰、PUA、系统性孤立、持续贬低等有害人际环境。
- 黄灯：被动死亡愿望或明确“撑不住”等安全线索，只做一次最小化确认，不显示红色危机。
- 红灯：明确本人当前自伤/自杀/伤人意图；已采取行为、无法保证安全或主动披露迫近危险时提高紧迫度。

### 8.2 红灯二级/三级

| 等级 | 触发 | 动作 |
|---|---|---|
| 黄灯（0） | 被动死亡愿望或明确“撑不住”，无当前伤害意图 | 继续倾听，只问一次当前是否有伤害想法；允许有/没有/不确定，不索要细节 |
| 二级（2） | 明确本人当前自杀、自伤或伤人意图 | 暂停普通建议但继续倾听，建议立即联系可信任真人和专业支持 |
| 三级（3） | 已采取伤害行为、无法保证安全，或主动披露迫近危险 | 建议立即联系真人、当地紧急服务和危机资源；不追问实施细节，不虚构已上报 |

当前危机资源包括：

- 中国全国统一心理援助热线：`12356`
- 学校心理咨询中心、辅导员、家人或紧急服务

### 8.3 入口、复核、出口与历史清洗

1. **入口拦截：** 用户最后一条消息先经过 `evaluate_safety()`。
2. **c3 复核：** 主回复生成后由 `safety_review()` 审核是否越界、遗漏危机、正常化有害环境或给出危险建议。
3. **出口兜底：** 绿灯回复若自行出现“你安全吗、无法保密、24 小时热线、紧急服务”等过度转介话术，确定性替换为普通陪伴回应。
4. **历史清洗：** 前端和后端都过滤旧危机 AI 回复与客户端 `system` 消息，避免下一轮被历史危机话术锚定。
5. **版本握手：** 前端要求后端返回当前契约版本；旧 Python 进程不会收到用户消息，而是提示重启服务。
6. **上游连接：** `httpx.AsyncClient(trust_env=False)` 绕过本机失效代理；OpenRouter 默认使用已验证可用的 `deepseek/deepseek-v4-flash`，错误写入服务日志。
