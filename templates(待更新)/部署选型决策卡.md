## 3. Railway 项目配置

1. 使用 GitHub 登录 Railway，并尽量完成账号验证：`https://railway.com/verify`。
2. New Project → Deploy from GitHub repo → 选择 `BNU-AI-Agent-Class/6`。
3. Service Settings → Root Directory 填 `/backend`。
4. 若 Railway 未自动读取子目录配置，在 Config File 中填 `/backend/railway.toml`。
5. Variables 按《密钥环境变量清单》配置；密钥不能写入代码。
6. 部署成功后，在 Settings → Networking 点击 Generate Domain。

`backend/railway.toml` 已固定：

```toml
[build]
builder = "RAILPACK"

[deploy]
startCommand = "uvicorn main:app --host 0.0.0.0 --port $PORT"
healthcheckPath = "/api/health"
healthcheckTimeout = 100
restartPolicyType = "ON_FAILURE"
restartPolicyMaxRetries = 5
```

## 4. 公网接口与验收标准

| 检查项 | 目标 |
|---|---|
| `GET /api/health` | HTTP 200；`status="ok"`；`version="3.3.0"`；`has_key=true` |
| `POST /api/chat` 普通输入 | HTTP 200；返回 `reply`，不误触发危机转介 |
| `POST /api/chat` 明确危机输入 | 确定性安全入口生效，返回对应红灯与转介信息 |
| CORS | 允许 `https://bnu-ai-agent-class.github.io`，不把密钥暴露给前端 |
| 前端联通 | 后续 `ship-golive` 把前端 API 地址改成 Railway HTTPS 域名 |

## 5. 试用期与平台风险

- Trial 为一次性 $5、最长 30 天；需要在 Railway Usage 页面观察实际消耗。
- Full Trial 依赖 Railway 对 GitHub 账号的自动验证；Limited Trial 有出站网络限制，可能无法访问 OpenRouter。
- 若 `/api/health` 正常但普通聊天调用失败，优先检查 Railway 验证状态、OpenRouter 出站访问和 Variables。
- 服务不使用本地持久化；对话历史仍由客户端请求携带。
- 试用到期前决定切换 Free 或 Hobby，避免交作业网址停止服务。
