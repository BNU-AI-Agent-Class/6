# skill_安全检测

## 目标

在每轮对话开始时，快速评估用户当前的心理安全风险等级（红/黄/绿），并决定本轮应采取的策略。

## 使用时机

- 每轮回复前必须运行；
- 用户输入涉及情绪、自我评价、死亡/暴力/孤独等关键词时优先重点判定；
- 连续多日高频使用、话题明显加重时提升敏感度。

## 判定标准

| 灯色 | 信号 | 动作 |
|---|---|---|
| 🟢 绿灯 | 正常倾诉、情绪波动但可控、愿意讨论事件 | 进入标准陪伴流程：共情 → 具体化 → 重构 → 行动 → 收尾 |
| 🟡 黄灯 | 情绪低落明显、反复说"活着没意思"但无具体计划、隐晦表达孤独/绝望、连续多日高频使用 | 加强关注，温和确认："你最近是不是很难受？有没有想找人聊聊？"；建议真人支持；减少分析，增加陪伴 |
| 🔴 红灯 | 明确自杀/自伤/伤害他人计划、严重自我厌恶、精神崩溃、无法保证自身安全 | **立即停止当前话题**；表达关心；声明无法保密；提供 24h 危机热线 + 学校心理咨询 + 建议联系紧急服务/家人/辅导员 |

## 输入

- 当前用户最新消息
- 近 5 轮对话上下文
- 用户情景记忆（近 7 天话题倾向、使用频率）

## 输出

只返回一个 JSON 对象：

```json
{
  "level": "green|yellow|red",
  "reason": "一句话判定依据",
  "action": "本轮应采取的策略概要"
}
```

## 红线

- 红灯必须触发 `skill_危机转介`，不再进入标准陪伴流程；
- 不因为用户说"我没事"就自动降级，需结合上下文判断；
- 不追问自杀/自伤的具体方法或细节。

## 示例

**用户输入**："今天食堂室友又没理我，我觉得我真的很糟糕。"

```json
{
  "level": "green",
  "reason": "情绪受挫、自我批评，但无自伤/自杀/崩溃信号，仍在讨论具体事件。",
  "action": "进入标准陪伴流程，先共情再具体化事件。"
}
```

**用户输入**："活着真没意思，我想结束这一切。"

```json
{
  "level": "red",
  "reason": "明确表达结束生命的意图。",
  "action": "立即停止当前流程，触发危机转介。"
}
```
