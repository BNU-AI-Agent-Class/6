# skill_安全检测

## 目标

每轮只根据“用户最新消息 + 用户历史消息”判断安全等级。Python 后端的确定性判定是最终依据，模型不得自行把绿灯升级为红灯。

## 判定总原则

- 关键词只是线索，必须结合说话主体、当前意图、否定语境和具体计划。
- 不扫描 AI 自己的回复来推断用户风险。
- 讨论课程、新闻、规则、第三方案例，或明确否认本人意图，不属于本人红灯。
- 历史中曾出现危机，不代表用户之后的普通问候仍是危机；每轮以最新用户消息为主。

## 四级标准

### 绿灯：普通陪伴

包括正常问候、倾诉、请求梳理情绪、低落、孤独、社交受挫、普通或严重自我批评，以及以下容易误判的表达：

- “你好，你能帮我梳理一下情绪吗？”
- “我今天参加了自杀预防讲座。”
- “我没有想自杀，只是最近情绪很乱。”
- “我的室友说他想自杀，我该怎么帮助他？”
- “我现在情绪有点低落，小组讨论时又没有人可以讨论。”
- “我觉得自己真的很糟糕。”
- “在你的梳理下，我更有信心了，现在有点小激动。”

动作：进入标准陪伴流程。不得询问“你现在安全吗”，不得声明无法保密，不得主动给危机热线或紧急服务。

### 黄灯：加强关注，不标红

信号：被动死亡愿望或明确“撑不住”等安全线索，但没有表达本人当前伤害意图。

示例：“活着没意思”“想消失”“我撑不下去了”。

动作：继续倾听，只问一次“你此刻有没有正在想伤害自己或结束生命？”用户可回答有/没有/不确定，不要求时间、地点、方法或计划细节。返回 `safety_light: yellow`、`is_crisis: false`，不展示红色危机气泡。

### 橙灯：有害人际环境

信号：霸凌、歧视、性骚扰、PUA、系统性孤立或持续贬低。

动作：不正常化，说明这超出普通摩擦，提供学校或可信任真人支持。橙灯不等于自杀危机。

### 红灯二级：明确当前意图

信号：用户明确表达本人当前想自杀、自伤或伤害他人，但尚未给出迫近计划。

示例：“我现在想自杀”“我想伤害自己”。

动作：停止普通分析，表达关心，建议立即联系可信任真人和专业支持。返回 `safety_light: red`、`is_crisis: true`、`escalation_level: 2`。

### 红灯三级：迫近危险（内部判定）

信号：用户主动披露可立即执行的计划或可用手段、已经采取伤害行为、明确无法保证自身或他人安全。计划信息会提高紧迫度，但不是获得倾听或支持的门槛。

示例：“我今晚打算跳楼”“我已经吞了很多药”“我无法保证自己安全”。

动作：继续简短倾听，暂停普通认知重构；建议立即联系身边真人、当地紧急服务和危机资源。不得追问实施细节。返回 `escalation_level: 3`。

## 输出

```json
{
  "level": "green|yellow|orange|red",
  "escalation": 0|2|3,
  "reason": "判定依据",
  "action": "本轮策略"
}
```

## 禁止事项

- 不因“难受、情绪、学校故事、想聊聊”等普通词汇触发红灯。
- 不因旧的 AI 危机回复或系统热线提示触发下一轮红灯。
- 不把黄灯显示成红灯。
- 不在绿灯回复中主动输出热线、无法保密或紧急服务话术。
- 不把“给出时间、地点、方法”写成用户必须满足的红线门槛，不诱导用户补充伤害细节。
