# skill_记忆读写

## 目标

管理三类记忆（情景记忆、语义记忆、程序记忆），让陪伴具备连续性，同时通过化名/脱敏保护用户隐私。

## 记忆类型

### 1. 情景记忆（Episodic Memory）

记录这个用户具体发生过什么、上次聊到哪、什么回应对他有效。

**示例**：
> "小远，食堂事件，对'点头微笑'动作表示愿意尝试；对'你想太多了'类回应明显反感。"

**写入时机**：
- 每次对话结束后的总结；
- 用户明确反馈某个回应对他有用/没用；
- 用户尝试了一个社交小行动并反馈结果。

### 2. 语义记忆（Semantic Memory）

记录一般性知识，用于提升回应质量。

**示例**：
- CBT 认知三角
- 自我同情三要素
- 正常社交焦虑 vs 自卑逃避判断标准
- 低压力社交动作库（点头、微笑、轻抬手、简短问候）

**写入时机**：
- 用户频繁出现的主题模式；
- 经过验证有效的认知重构框架或动作；
- 用户明确认可的解释或概念。

### 3. 程序记忆（Procedural Memory）

记录一次陪伴的固定套路，确保流程稳定。

**示例**：
> 开场 → 共情 → 具体化事件 → 区分事实/想法 → 认知重构 → 判断是否需要转介 → 提取小行动 → 收尾退场。

**写入时机**：
- 首次对话建立用户画像后；
- 用户进入稳定陪伴节奏后；
- 每次对话结束后微调流程偏好。

## 读取规则

1. 每轮对话开始时读取全部三类记忆；
2. 读取时只使用化名或脱敏描述，不出现真实姓名、学号、具体地点等敏感信息；
3. 情景记忆优先读取最近 3 次对话摘要；
4. 语义记忆按当前话题匹配最相关的条目；
5. 程序记忆按用户当前阶段（首次/稳定/危机后）选择对应流程。

## 写入规则

1. 每轮对话结束后更新，避免写入未经用户确认的诊断/判断；
2. 情景记忆只写客观事实和用户原话反馈，不写推测；
3. 语义记忆写入经过验证的通用知识，标注来源/验证状态；
4. 程序记忆写入实际运行效果好的流程变体；
5. 所有记忆写入前进行隐私脱敏：真实姓名替换为化名，具体地点泛化。

## 输出格式

### 读取输出

```json
{
  "episodic": ["摘要1", "摘要2"],
  "semantic": ["相关知识1", "相关知识2"],
  "procedural": "当前适用流程"
}
```

### 写入输出

```json
{
  "updated": true,
  "summary": "本次更新内容的简要说明"
}
```

## 隐私边界

- 不存储用户真实姓名、学校、住址、联系方式等可识别信息；
- 不将记忆内容用于产品改进或分享给无关第三方；
- 用户要求删除记忆时，立即执行并确认。
